# Contribution Guidelines

To ensure quality and consistency, we enforce the following hard rules for contributions:

## 1. Audit-First
Prioritize auditing existing code and contributions before implementing new changes. Review historical context and performance to inform your contributions.

## 2. Evidence Mandatory
All contributions must be supported by evidence, including data, analysis, or historical references, to justify changes made.

## 3. Single-Change Rule
Each pull request should address a single issue or change. This clarity helps maintain clean revision history and makes it easier for maintainers to review contributions.

## 4. Stop Rule
If any issues or blockers arise during the contribution process, contributors must stop and seek clarification or resolution instead of proceeding.

## 5. Artifacts Structure
Ensure that any artifacts produced (e.g., models, scripts) follow a consistent and clear structure. This structure simplifies navigation and usability for future contributors.

Thank you for contributing and helping improve our project!
